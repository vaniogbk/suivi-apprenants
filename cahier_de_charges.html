1 Présentation du projet

1.1 Contexte et problématique

Les formateurs et enseignants gérant des groupes d'apprenants en formation professionnelle ou académique font face à un défi quotidien : enregistrer, suivre et analyser les présences de manière simple et rapide, sans dépendre d'un réseau ou d'un serveur tiers.

Les solutions existantes (tableurs, logiciels de gestion lourds, plateformes cloud) présentent plusieurs limites : complexité d'utilisation, dépendance internet, coût, ou manque de fonctionnalités d'analyse.

Problème central : Comment permettre à un formateur de faire l'appel, consulter l'historique des présences et générer des rapports, depuis n'importe quel navigateur, sans installation ni connexion serveur ?

1.2 Objectifs

  • Objectif principal : Fournir une interface web intuitive de suivi des présences, utilisable sans connexion à un serveur.
  • Objectif secondaire : Permettre l'analyse statistique de la présence et l'export de rapports (PDF, CSV).
  • Objectif tertiaire : Offrir une expérience soignée avec thème sombre, responsive design et animations fluides.

1.3 Public cible

👩‍🏫 Utilisateur principal

Formateur / Enseignant gérant 1 à 3 groupes de 10 à 50 apprenants. Utilise l'application quotidiennement pour faire l'appel.

👨‍💼 Utilisateur secondaire

Responsable pédagogique souhaitant consulter les statistiques de présence et exporter des rapports pour des bilans mensuels.

2.2 Module — Suivi de Présence

Module central permettant de réaliser l'appel quotidien par groupe.

Fonctionnalité Description Priorité
Sélection de date Saisie de la date via un champ de type date (par défaut : date du jour). P1 — Critique
Sélection de session/groupe Choix du groupe parmi les sessions configurées (ex. Marketing Digital, Dév. Web). P1 — Critique
Liste d'appel Affiche tous les apprenants du groupe sélectionné dans un tableau interactif. P1 — Critique
Statut de présence 3 statuts possibles par apprenant : Présent, Absent, Retard. Marquage en un clic. P1 — Critique
Note / Heure Champ libre pour saisir l'heure d'arrivée ou une note pour chaque apprenant. P2 — Important
Compteurs présents/absents Badges affichant en temps réel le nombre de présents et absents. P2 — Important
Enregistrement Sauvegarde des présences en localStorage via le bouton "Enregistrer". P1 — Critique

Règle métier :

  • Un retard est comptabilisé comme présent dans le calcul du taux de présence.
  • Les données de présence sont structurées par date → session → apprenant.
  • Il est possible de modifier une présence déjà enregistrée en rechargeant la même date.

2.3 Module — Gestion des Apprenants

Gestion complète (CRUD) des fiches apprenants.

Fonctionnalité Description Priorité
Grille des apprenants Affichage en grille responsive de cartes par apprenant (avatar, nom, groupe). P1 — Critique
Recherche dynamique Filtrage en temps réel par nom ou groupe, sans rechargement de page. P1 — Critique
Filtrage par groupe Menu déroulant pour n'afficher qu'un groupe spécifique. P2 — Important
Ajout d'apprenant Formulaire modal avec champs : Nom complet*, Email, Groupe, Photo (upload). P1 — Critique
Modification Pré-remplissage du modal avec les données existantes pour édition. P1 — Critique
Suppression avec confirmation Dialogue de confirmation avant suppression définitive d'un apprenant. P1 — Critique
Photo de profil Upload d'image, encodée en base64 et stockée localement. Avatar généré par initiales si absent. P3 — Souhaitable

Champs du profil apprenant :

  • id : Identifiant unique généré automatiquement
  • name : Nom complet (obligatoire)
  • email : Adresse email (optionnel)
  • group : Nom du groupe/session (optionnel, avec auto-complétion)
  • photo : Image encodée base64 (optionnel)
  • createdAt : Date de création (automatique)

3 Architecture technique

3.1 Choix technologiques

L'application adopte une approche zéro dépendance de framework pour maximiser la portabilité et minimiser les points de défaillance.

Composant Technologie Justification
Structure HTML5 Sémantique, accessible, universel
Style CSS3 Vanilla Variables CSS, transitions, responsive sans framework
Logique JavaScript ES6+ Vanilla Modules objet (singleton pattern), sans dépendance
Persistance localStorage Données persistantes côté client, sans serveur
Icônes Phosphor Icons Library légère, complète, chargée via CDN
Export PDF jsPDF 2.5.1 Génération PDF client-side, éprouvée
Capture HTML html2canvas 1.4.1 Rendu HTML vers image pour intégration PDF
Police Inter (Google Fonts) Lisibilité optimale pour interfaces de données

Chaque module est un objet singleton exposé globalement. L'ordre de chargement des scripts garantit les dépendances :

5 Données et persistance

5.1 Stratégie de stockage

L'application utilise exclusivement le localStorage du navigateur comme mécanisme de persistance. Aucun serveur, aucune base de données externe, aucune connexion réseau ne sont nécessaires pour le fonctionnement nominal.

Capacité : Le localStorage offre typiquement 5 à 10 Mo par domaine. Pour 48 apprenants avec photos base64 (≈ 50 Ko/photo), la limite peut être atteinte. Sans photos, les données textuelles de milliers d'apprenants restent dans les limites.

5.2 Initialisation

  • Première visite : Génération automatique de données de démonstration (48 apprenants, 2 groupes, 5 jours de présences).
  • Visite suivante : Chargement depuis localStorage avec validation de structure.
  • Données corrompues : Détection automatique, réinitialisation et régénération des données démo.

5.3 Sauvegarde et restauration

  • Export JSON : Téléchargement d'un snapshot complet nommé suivi_backup_YYYY-MM-DD.json
  • Import JSON : Restauration avec validation du format avant écrasement des données actuelles
  • Portabilité : Un fichier de sauvegarde peut être importé sur n'importe quel appareil/navigateur

5.4 Sécurité et confidentialité

Important : Les données sont stockées en clair dans le localStorage du navigateur. Il est déconseillé de stocker des données personnelles sensibles sans protection supplémentaire. L'application est à usage local — le formateur est responsable de la sécurité de son poste.

6 Contraintes et exigences non-fonctionnelles

6.1 Performance

  • Chargement initial en moins de 2 secondes sur connexion standard (bibliothèques CDN mises en cache).
  • Rendu des listes de 50+ apprenants sans délai perceptible.
  • Aucune latence réseau pour les opérations de lecture/écriture (tout en mémoire + localStorage).

6.2 Compatibilité navigateurs

  • Chrome 90+ / Edge 90+ / Firefox 88+ / Safari 14+
  • Aucun support requis pour Internet Explorer.
  • Fonctionnement hors-ligne une fois la page chargée (sauf rechargement des CDN).

6.3 Accessibilité

  • Navigation clavier fonctionnelle (focus visible, tabindex).
  • Contraste de couleurs conforme aux recommandations WCAG AA.
  • Labels explicites sur tous les champs de formulaire.

6.4 Expérience utilisateur

  • Retour visuel immédiat pour toute action (toast de confirmation).
  • Double confirmation pour les actions destructives (suppression, réinitialisation).
  • Animation de transition pour les changements d'état (fade-in, slide-in).
  • Mode sombre disponible pour usage en environnement peu éclairé.

6.5 Maintenabilité

  • Code organisé en modules distincts à responsabilité unique (SRP).
  • Mécanisme de migration de données prévu pour les évolutions futures du schéma.
  • Variables CSS centralisées pour faciliter la personnalisation du thème.

8 Glossaire

Terme Définition
Apprenant Étudiant ou stagiaire inscrit dans une formation, dont on suit la présence.
Session Groupe d'apprenants suivant une même formation (ex. "Marketing Digital").
Présent Apprenant arrivé dans les délais lors d'une journée de formation.
Absent Apprenant non présent lors d'une journée de formation.
Retard Apprenant arrivé après l'heure prévue. Compté comme présent dans le taux global.
Taux de présence Ratio (Présents + Retards) / Total des enregistrements, exprimé en pourcentage.
KPI Key Performance Indicator — indicateur clé de performance affiché sur le tableau de bord.
SPA Single Page Application — application web dont la navigation se fait sans rechargement complet de la page.
localStorage Mécanisme de stockage côté navigateur, persistant entre les sessions, limité à 5-10 Mo.
Base64 Format d'encodage d'images binaires en texte, permettant le stockage dans localStorage.
