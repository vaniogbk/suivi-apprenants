🚀 Étape 1 — Mise en place du projet Figma

1

Créer un nouveau fichier Figma

Sur figma.com → New Design File → Nommer : Suivi Apprenants — Maquette v1.0

2

Configurer les frames Desktop

Pour chaque écran, créer un Frame (F) de taille 1440 × 900px (Desktop HD).
Nommer chaque frame : 01 — Dashboard, 02 — Présence, 03 — Apprenants, 04 — Statistiques, 05 — Paramètres

3

Ajouter la police Inter

Dans Figma, la police Inter est disponible nativement. Aller dans le panneau texte et sélectionner "Inter" avec les poids 400, 500, 600, 700.

4

Installer le plugin Phosphor Icons

Menu → Plugins → Browse → Chercher "Phosphor Icons" → Installer. Ce plugin donne accès aux icônes utilisées dans l'app.

5

Organiser les pages Figma

  • Page 1 : 🎨 Design System (couleurs, typographie, composants)
  • Page 2 : 💻 Écrans Desktop (5 vues)
  • Page 3 : 📱 Écrans Mobile (optionnel)
  • Page 4 : 🌙 Thème Sombre (optionnel)

Détails de construction :

  • Sidebar : Rectangle blanc 280×900px, border-right #E5E7EB 1px. AutoLayout vertical, gap 8px, padding 32px 20px.
  • Logo : Icône 32×32px bg dégradé indigo (radius 8px) + texte "Suivi Apprenants" Inter 700 18px couleur #4F46E5.
  • Nav item actif : Fond #EEF2FF, texte #4338CA, radius 8px.
  • StatCards : AutoLayout horizontal entre les 3 cartes, gap 24px. Chaque carte radius 12px, ombre "Shadow/SM" (0 1px 2px rgba(0,0,0,0.05)). Barre gauche 4px bg dégradé indigo.
  • Carte "Aujourd'hui" : Full width, radius 12px, padding 24px. En-tête flex space-between avec bouton primaire.
  • Fond global : #F9FAFB (gris très clair)

Donut Chart (graphique circulaire) :

  • Cercle extérieur : 150×150px, couleur selon taux (vert si ≥80%, orange si ≥50%, rouge sinon)
  • Cercle intérieur (fond blanc) : 120×120px, centré — affiche le pourcentage + label
  • Dans Figma : utiliser 2 cercles empilés avec des arcs, ou le plugin "Arc" pour créer un donut

Barre de progression dans le tableau :

  • Fond gris 100px × 6px, radius 3px
  • Remplissage coloré selon le taux (vert/orange/rouge), largeur = taux%
  • Valeur numérique à droite de la barre

📱 Écran Bonus — Version Mobile

Créer des frames 390 × 844px (iPhone 14) pour les 3 écrans principaux.

Sur mobile, la sidebar devient un menu hamburger en bas de l'écran (Bottom Navigation Bar) ou un drawer glissant depuis la gauche.

Adaptations principales :

  • Navigation : Bottom Tab Bar de 64px avec 5 icônes (Dashboard, Présence, Apprenants, Stats, Menu)
  • Dashboard : KPIs en colonne (1 par ligne), full width
  • Apprenants : Grille 1 colonne, cards full width
  • Présence : Tableau scrollable horizontalement, actions simplifiées
  • Statistiques : Donut chart centré + tableau scrollable

🔗 Étape finale — Créer le Prototype Figma

1

Activer le mode Prototype

Panneau droit → Onglet "Prototype" (icône triangle lecture)

2

Connecter les nav items

Pour chaque bouton de navigation dans la Sidebar, créer une connexion vers le frame correspondant :
"Dashboard" → frame 01 | "Présence" → frame 02 | "Apprenants" → frame 03 | "Stats" → frame 04 | "Paramètres" → frame 05

Transition : Smart Animate, durée 200ms, easing "Ease out"

3

Connecter les boutons d'action

  • Bouton "Faire l'appel" (Dashboard) → frame 02 Présence
  • Bouton "+ Ajouter" (Apprenants) → Modal Ajouter apprenant (overlay)
  • Bouton "Enregistrer" (Présence) → Toast de confirmation (variant success)
4

Tester et partager

Présenter → Lecture ▶ pour naviguer le prototype.
Partager le lien "View only" pour la revue client/jury.
